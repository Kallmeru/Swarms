# SWARMS: frontend

Static site, no build step. Reads everything from `web/data/` and animates the swarm graph.

## Run locally
```
python -m http.server 8000 --directory web
```
then open http://localhost:8000

## Data contract
Every file in `web/data/` is generated by `benchmark/run_benchmark.py` (core + attack-lab branches):
- `manifest.json`: list of attacks for the dropdown
- `benchmark_summary.json`: aggregate success rates for the bar chart
- `<attack_id>_off.json` / `<attack_id>_on.json`: ordered event arrays to animate, one pair per attack

The two attacks currently in `web/data/` (`attack_001`, `attack_002`) are **hand-written placeholder fixtures** for frontend development, not real benchmark output. They'll be replaced automatically once the real benchmark script runs and writes into this folder. The event schema is documented in full in `docs/swarms-briefs/person3-frontend-visualization.md` (also covers the shared contracts with the core and attack-lab parts).

Nothing in `web/` calls an LLM or executes anything. It is a pure read-only visualization of pre-generated JSON.
